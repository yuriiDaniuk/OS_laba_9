# OS_laba_9
Даний проект реалізує модель взаємодії "Науковий керівник" (Сервер) та "Виконавці" (Клієнти/Студенти), які спільно генерують ідеї для проекту протягом обмеженого часу. Взаємодія між процесами (програмами) реалізована за допомогою **сокетів** (Варіант 6).

**Основні етапи моделювання:**
1.  **Керівник** (Сервер) запускає "Віртуальну дошку" (файл) та очікує підключення **Виконавців** (Клієнтів).
2.  **Виконавці** підключаються та генерують ідеї, надсилаючи їх Серверу.
3.  Через **3 хвилини** Керівник зупиняє генерацію, збирає всі ідеї та виводить їх.
4.  **Виконавці** голосують за 3 найкращі ідеї.
5.  **Керівник** фіналізує роботу, записує 3 найкращі ідеї на дошку та зберігає вміст файлу.

### Інструкції з Запуску

Оскільки проект використовує сокети, необхідно спочатку запустити **Сервер (Керівник)**, а потім **Клієнтів (Виконавці)**.

#### Крок 1: Запуск Сервера (Керівник)

1.  Перейдіть до директорії `server_panel/`.
2.  Запустіть сервер, вказавши **HOST** (наприклад, `127.0.0.1` для локальної мережі) та **PORT** (наприклад, `65432`).

    ```bash
    # Приклад для Python
    python server.py 127.0.0.1 65432
    # Або
    ./server_panel/server 127.0.0.1 65432
    ```
    Сервер виведе повідомлення: `Сервер запущено. Очікування підключення виконавців на 127.0.0.1:65432...`

#### Крок 2: Запуск Клієнтів (Виконавці)

1.  Відкрийте нові термінали для кожного виконавця (рекомендовано 3-4 клієнти).
2.  Перейдіть до директорії `client_panel/`.
3.  Запустіть клієнта, використовуючи ті ж самі **HOST** та **PORT**.

    ```bash
    # Приклад для Python
    python client.py 127.0.0.1 65432
    # Або
    ./client_panel/client 127.0.0.1 65432
    ```
    Клієнт виведе: `Підключено до Сервера. Можна генерувати ідеї.`

> **Примітка:** На час генерації ідей (3 хвилини), клієнти можуть надсилати текстові повідомлення (ідеї) через консоль. Сервер автоматично збереже їх та перейде до етапу голосування.

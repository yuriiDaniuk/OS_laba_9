# OS_laba_9
Даний проект реалізує модель взаємодії "Науковий керівник" (Сервер) та "Виконавці" (Клієнти/Студенти), які спільно генерують ідеї для проекту протягом обмеженого часу. Взаємодія між процесами (програмами) реалізована за допомогою **сокетів** (Варіант 6).

**Основні етапи моделювання:**
1.  **Керівник** (Сервер) запускає "Віртуальну дошку" (файл) та очікує підключення **Виконавців** (Клієнтів).
2.  **Виконавці** підключаються та генерують ідеї, надсилаючи їх Серверу.
3.  Через **3 хвилини** Керівник зупиняє генерацію, збирає всі ідеї та виводить їх.
4.  **Виконавці** голосують за 3 найкращі ідеї.
5.  **Керівник** фіналізує роботу, записує 3 найкращі ідеї на дошку та зберігає вміст файлу.

### Інструкції з Запуску

Оскільки проект використовує сокети, необхідно спочатку запустити **Сервер (Керівник)**, а потім **Клієнтів (Виконавці)**.
1. Для того, щоб працювало, потрібно всім користувачам бути під'єднаним до **одної локальної мережі**
2. Керівник повинен дізнатися через `ipconfig` IP-адресу та поділитися з користувачами.
3. Користувачі мають підключитися до нього та отримати повідомлення про успіх підключення.

#### Крок 1: Запуск Сервера (Керівник)

1.  Запустіть програму
2.  Дізнайтеся IP-адресу командою `ipconfig`
3.  Зачекайте допоки всі користувачі під'єднаються
4.  Почніть *Topic*
5.  Очікуйте введення ідей протягом 3хв або завершіть введення вручну
6.  Почніть голосування
7.  Очікуйте поки пройде голосування та виведіть результати.

#### Крок 2: Запуск Клієнтів (Виконавці)

1.  Запустіть програму
2.  Під'єднайтеся до однієї локальної мережі, та отримайте від керівника IP-адресу
3.  Встановть з'єднання з сервером під своїм ніком
4.  Очікуйте початок часу для генерації ідей (3хв), відповідно введіть їх
5.  Проголосуйте за найкращу ідею
6.  Перегляньте результати голосування
